Название тест-кейса
Интеграционный тест: передача данных из Системы A в Систему B через API.

Цель теста
Проверить корректность передачи данных из Системы A в Систему B через API, включая обработку данных, сохранение в целевой системе и соответствие форматов.

Описание
Тест проверяет интеграцию между двумя системами, где Система A отправляет данные (например, заказ) в Систему B через REST API. Проверяемые аспекты:

Успешность HTTP-запроса.

Корректность данных в Системе B после интеграции.

Обработка ошибок (при невалидных данных или сбоях связи).

Предусловия
Система A и Система B доступны и настроены для интеграции.

Пользователь авторизован в обеих системах.

Тестовые данные подготовлены (например, заказ в Системе A).

Тестовые данные
Пример заказа в Системе A:

json
Copy
{
  "order_id": "TEST-123",
  "amount": 1500.00,
  "customer_email": "test@example.com",
  "items": ["Item1", "Item2"]
}
Шаги теста
Шаг 1: Создание тестового заказа в Системе A
Войдите в Систему A.

Создайте новый заказ с данными, указанными выше.

Убедитесь, что заказ успешно сохранен в Системе A.

Ожидаемый результат: Заказ отображается в списке заказов Системы A с статусом "Ожидает отправки".

Шаг 2: Отправка данных из Системы A в Систему B
Запустите процесс интеграции (вручную или через триггер).

Дождитесь завершения передачи данных.

Ожидаемый результат:

Система A возвращает статус 200 OK или сообщение об успешной отправке.

В логах Системы A нет ошибок.

Шаг 3: Проверка данных в Системе B
Войдите в Систему B.

Найдите заказ с ID TEST-123 через поиск или в списке заказов.

Сравните данные заказа в Системе B с тестовыми данными из Системы A.

Ожидаемый результат:

Заказ с ID TEST-123 существует в Системе B.

Поля amount, customer_email, items совпадают с исходными данными.

Шаг 4: Проверка обработки ошибок (негативный сценарий)
В Системе A создайте заказ с невалидными данными (например, пустой customer_email).

Попытайтесь отправить данные в Систему B.

Ожидаемый результат:

Система A возвращает статус 4xx или сообщение об ошибке валидации.

Заказ не появляется в Системе B.

Шаг 5: Проверка повторной отправки данных
Повторите Шаг 2 для уже отправленного заказа TEST-123.

Ожидаемый результат:

Система B возвращает ошибку 409 Conflict или игнорирует дубликат.

В Системе B остается только одна запись TEST-123.

Пост-условия
Удалите тестовый заказ TEST-123 из Системы A и Системы B.

Очистите логи (если требуется).

Критерии успеха
Все шаги позитивного сценария завершены с ожидаемыми результатами.

Ошибки в негативных сценариях обрабатываются корректно.
